# Архитектура системы

## Общая схема

Система построена по монолитной веб-архитектуре:

- браузер пользователя;
- Django backend;
- PostgreSQL как основная СУБД;
- файловое хранилище для загруженных материалов;
- встроенная административная панель Django для редактора и администратора.

## Почему выбран такой подход

- простая поддержка и развертывание;
- естественная интеграция с PostgreSQL;
- устойчивый серверный рендеринг;
- низкий порог сопровождения после защиты;
- отсутствие необходимости в отдельном фронтенд-сервисе.

## Подсистемы

- `accounts`: роли, профили, действия пользователей.
- `corpus`: журнал, выпуски, статьи, тексты, токены, леммы, импорт, поиск, подкорпуса.
- `analytics`: частотные таблицы, биграммы, сравнение подкорпусов, графики.
- `core`: главная страница, общие шаблоны и навигация.
