# Как работает импорт, поиск и подкорпуса

## Импорт

Система поддерживает два режима пополнения:

1. Ручной ввод через административный интерфейс.
2. Пакетный импорт из каталога с `metadata.json` и файлами.

Во втором случае для каждой статьи создаются:

- выпуск;
- авторы и аффилиации;
- карточка статьи;
- очищенный текст;
- файл публикации;
- токены и леммы для последующего поиска.

## Поиск

- полнотекстовый поиск и поиск по фразе используют `PostgreSQL Full Text Search`;
- поиск по словоформе и лемме использует `ArticleToken` и `Lemma`;
- метаданные фильтруются отдельно и могут сочетаться с любым поисковым режимом.

## Подкорпуса

Подкорпус сохраняет набор фильтров и автоматически формирует состав статей.
Он может использоваться как отдельная база для частотного анализа, биграмм и сравнения с другим подкорпусом.

## География авторов

Аффилиации авторов дополнительно проходят географическую нормализацию. Система пытается определить город по названию организации, привязывает его к сущности `CityLocation` и сохраняет координаты. Это позволяет строить карту авторской географии без ручного ввода координат для каждой статьи.

Основной безопасный режим:

```text
python manage.py refresh_author_geography
```

Для новых городов, которых нет во встроенном справочнике, предусмотрен расширяемый режим через внешний геокодер Nominatim:

```text
python manage.py refresh_author_geography --geocode
```

Внешние результаты сохраняются в базе и помечаются как требующие редакторской проверки, поэтому изменение сайта журнала не ломает уже накопленную географическую базу.
