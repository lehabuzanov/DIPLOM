# Инструкция редактора и администратора

## Роли

### Исследователь

- выполняет поиск;
- сохраняет запросы;
- сохраняет подкорпуса;
- использует аналитику.

### Редактор

- загружает статьи вручную;
- редактирует записи через интерфейс и Django admin;
- запускает импорт и восстановление текстов.

### Администратор

- управляет пользователями и ролями;
- контролирует целостность корпуса;
- запускает служебные команды;
- проверяет корректность развертывания.

## Ручная загрузка статьи

Пользовательская форма:

```text
/corpus/articles/upload/
```

Через нее можно:

- добавить новую статью;
- привязать выпуск;
- указать авторов и аффилиации;
- загрузить PDF или DOCX;
- вставить очищенный текст;
- заполнить DOI, аннотацию и ключевые слова.

## Административная панель

```text
/admin/
```

Через Django admin можно управлять:

- выпусками;
- статьями;
- авторами;
- разделами;
- файлами;
- текстами;
- сохраненными запросами;
- подкорпусами;
- пользователями и ролями.

## Импорт архива журнала

Основная команда синхронизации:

```powershell
docker compose exec web python manage.py sync_ojs_journal --skip-existing
```

Без Docker:

```powershell
python manage.py sync_ojs_journal --skip-existing
```

Что делает команда:

- обходит архив OJS журнала;
- добавляет новые выпуски и статьи;
- загружает PDF;
- извлекает текст;
- создает или обновляет метаданные;
- запускает индексацию через сохранение `ArticleText`.

## Автодобавление новых статей

Для серверного внедрения рекомендуется настроить регулярный запуск:

```powershell
python manage.py sync_ojs_journal --skip-existing
```

Рекомендуемый режим:

- `1` раз в сутки или после выхода нового выпуска;
- через `cron`, `systemd timer` или планировщик задач Windows;
- с логированием результата выполнения.

Ограничение:

- синхронизация зависит от структуры сайта журнала на OJS;
- если OJS-тема или верстка страниц заметно изменятся, потребуется корректировка парсера.

## Восстановление текстов

Если часть статей импортировалась без текста, выполните:

```powershell
docker compose exec web python manage.py repair_article_texts
```

Команда:

- берет локальные PDF, если они есть;
- при необходимости скачивает PDF по внешнему URL;
- восстанавливает `ArticleText`;
- автоматически перестраивает токены, леммы и поисковый индекс.

## Переиндексация корпуса

```powershell
docker compose exec web python manage.py rebuild_corpus_index
```

Используйте после массовых изменений текстов или правил токенизации.

## Самопроверка

```powershell
docker compose exec web python manage.py selfcheck_corpus
```

Проверяются:

- главные страницы;
- поиск;
- аналитика;
- карточка статьи;
- доступ к файлу статьи;
- полнота текстов;
- лемматический и полнотекстовый поиск;
- кабинет пользователя;
- список сохраненных запросов и подкорпусов;
- редакторская форма загрузки.

## Рекомендованный порядок обслуживания

1. Запустить сайт и БД.
2. Выполнить `sync_ojs_journal --skip-existing`.
3. Выполнить `repair_article_texts`.
4. Выполнить `selfcheck_corpus`.
5. При необходимости выполнить `rebuild_corpus_index`.
